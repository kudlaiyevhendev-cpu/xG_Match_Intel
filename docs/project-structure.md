## Generated / Runtime Artifacts

| Artifact | Location | Notes |
|---|---|---|
| OpenAPI spec | `/v3/api-docs` (JSON), Swagger UI at `/swagger-ui.html` | Generated by springdoc at runtime; optionally exported to `docs/openapi.json` in CI |
| Docker image | GHCR (or local) | Built in CI via multi-stage Dockerfile |
| Flyway schema history | `flyway_schema_history` table | Managed automatically |
| Agent traces | `agent_run` table + structured logs | Demo-worthy: shows tool-call chains |
| Cached metrics/reports | Redis (`metrics:*`, `report:*`, `llm:*` keys) | TTL + data-version invalidation |
| Test reports / coverage | `target/site/jacoco` | Published as CI artifact |
| Seed data | `data/seed/` (small StatsBomb subset) | Lets `docker compose up` demo work offline/fast |

## Conventions

- **Branching**: trunk-based, short-lived feature branches, PRs even solo (CI gate + reviewable history for portfolio).
- **Commits**: conventional commits (`feat:`, `fix:`, `chore:`) — clean log reads well to reviewers.
- **ADRs**: every non-obvious choice (Postgres job queue instead of Kafka, in-process tools instead of HTTP loopback) gets a one-page ADR in `docs/adr/`.
- **No layer skipping**: controllers → services → repositories; agent tools call services, never repositories.
